# Implementation Checklist

## Objective

Provide a phased checklist for standing up a nonprofit fundraising operations model and supporting CRM readiness.

## Phase 1: Discovery

- define business goals
- identify key fundraising processes
- document current tools and spreadsheets
- identify reporting pain points
- confirm key stakeholders and owners
- inventory current campaigns, events, and donor segments

## Phase 2: Process Design

- map current and future state workflows
- define donor lifecycle stages
- define sponsorship workflow
- define acknowledgement and stewardship rules
- establish revenue definitions and campaign attribution rules
- document approval points and handoffs

## Phase 3: Data Preparation

- inventory donor, sponsor, and campaign data sources
- identify duplicate risks
- standardize naming conventions
- define required fields
- map legacy fields to target structure
- identify missing historical data

## Phase 4: CRM Configuration Planning

- define account and contact model
- define opportunity types and stages
- define campaign hierarchy
- define dashboard requirements
- define role based ownership needs
- define required automations and validations

## Phase 5: Pilot

- select one campaign or event for pilot use
- validate workflow in live conditions
- test gift entry and reporting outputs
- test sponsorship tracking and fulfillment visibility
- confirm acknowledgment timing
- capture lessons learned

## Phase 6: Training

- train staff on data entry standards
- train leadership on pipeline review expectations
- train finance support on reconciliation handoffs
- publish quick reference guides
- confirm escalation path for questions

## Phase 7: Reporting Rollout

- validate dashboard definitions
- reconcile key totals before launch
- publish audience specific dashboards
- establish refresh cadence
- define dashboard owners

## Phase 8: Governance Handoff

- launch recurring operating cadence
- finalize issue and risk log ownership
- establish change control process
- assign documentation owner
- schedule periodic process review

## Exit Criteria

Implementation should not be considered complete until:
- core workflows are documented
- required data standards are enforced
- key reports are usable
- stakeholders know their responsibilities
- reconciliation process is defined and operating
